Cross-Region Communication
Submit & Moderate connect to RabbitMQ in East Asia via public IP over AMQP (port 5672). Submit fetches joke types from Joke Service API.

Data Flow: Joke Lifecycle

1
Browse: User visits /joke-ui via Kong. Kong routes to Joke Service. Joke API queries MySQL and returns random jokes.
2
Submit: User submits joke via /submit-ui. Submit Service publishes joke to RabbitMQ submit_joke queue (cross-region AMQP).
3
Moderate: Moderator logs in via Auth0 OIDC at /moderate-ui. Moderate Service consumes from submit_joke queue. Moderator approves/rejects.
4
ETL: Approved jokes are published to moderated_joke queue. ETL Worker in East Asia consumes and inserts into MySQL database.

Security Hardening Measures

NSG Firewall Rules — Service ports restricted to specific VM IPs. Only Kong accepts public HTTP/HTTPS. Explicit deny-all rule.
RabbitMQ Auth — Custom credentials via GitHub Secrets. Guest user disabled for remote access. Connection limits enforced.
Kong Gateway — Rate limiting on all services. CORS restricted to gateway origin. Security headers (XSS, CSRF, Clickjacking). Request size limits.
Auth0 OIDC — Moderate service protected by OpenID Connect. Secrets stored in GitHub Actions, never in code. Session-based authentication.
Production improvements: Azure VNet Peering (private cross-region), Azure Bastion (SSH management), TLS everywhere, Azure Key Vault for secrets
